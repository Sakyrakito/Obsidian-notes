Дан неориентированный граф и n вершин и m рёбер. Нужно найти количество треугольников (циклов длинной 3).

## Способ 1:
Пройти по все тройкам вершин и проверить, что они делают цикл. 
Асимптотика $O(n^3)$

## Способ 2:
Для каждого ребра $(u, v)$ найти общих соседей $u$ и $v$.
То есть:

$$
\large N(u) \cap N(v)
$$

Каждая вершина из пересечения даёт треугольник, но если степени вершин большие, то асимптотика становится большой, а именно:

$$
\large \sum_{(u, v) \in E}{min(deg(u), deg(v))}
$$

## Способ 3:
Заметим, что в графе не может быть много вершин с большой степенью.
Поэтому поделим вершины на 2 типа:
1. Лёгкие (light) - вершины с маленькой степенью
2. Тяжёлые (heavy) - вершины с большой степенью

Пусть 

$$
B = \sqrt{ m }
$$

где $m$ - число рёбер.

Теперь:
если $deg(v) < B \to light$
если $deg(u) \geq B \to heavy$

#### Почему вершин heavy мало?
Всего у нас в графе $2m$ вершин, а значит тяжёлых всего:

$$
\large \frac{2m}{B}
$$

Подставим $B = \sqrt{ m }:$

$$
\large \frac{2m}{\sqrt{ m }} = 2\sqrt{ m }
$$

#### Идея
Будем искать треугольники двумя способами.
Разделим их на случаи:
1. треугольники через light вершины
2. треугольники через heavy вершины

#### 1 случай
1. В цикле нет тяжелых вершин. Рассмотрим какое-нибудь ребро $(a,b)$ цикла. Третья вершина $c$ должна лежать в объединении списков смежности $g_{a}$​ и $g_{b}​$, а раз обе эти вершины лёгкие, то таких вершин найдётся не более $n$. Значит, всего циклов этого типа может быть не более $O(m\sqrt{ n }​)$.
2. В цикле одна тяжелая вершина. Аналогично — есть одно «лёгкое» ребро, а значит таких циклов тоже $O(m​\sqrt{ n })$.

#### 2 случай
1. В цикле две тяжелые вершины — обозначим их как $a$ и $b$, а лёгкую как $c$. Зафиксируем пару $(a,c)$ — способов это сделать $O(m)$, потому что всего столько рёбер. Для этого ребра будет не более $O(n)$ рёбер $(a,b)$, потому что столько всего тяжелых вершин. Получается, что всего таких циклов может быть не более $O(m\sqrt{ n })$.
2. Все вершины тяжелые. Аналогично — тип третьей вершины в разборе предыдущего случая нигде не использовался; важно лишь то, что тяжелых вершин bb немного.

Получается, что циклов длины 3 в графе может быть не так уж и много — не более $O(m\sqrt{ n })$.

## Самая популярная версия алгоритма
Это более удобная форма 3 способа.
Она называется **degree ordering.**

Для каждого ребра $v, u$:
направим его от **меньшей степени к большей**.

$$
\large v \to u\ если\ deg(v) < deg(u)
$$

если степени равны:

$$
\large v \to u\ если\ v < u
$$

После такой ориентации у каждой вершины будет мало исходящих рёбер.
Максимум примерно $\sqrt{ m }$.

Теперь нужно найти:

$$
\large v \to u \to w, v \leq u \leq w
$$

И проверять существование ребра $v \to w$.

Теперь алгоритм:

Для каждой вершины $v$ пометим всех её исходящих соседей $mark[u] = true$.
Для каждого соседа $u$ смотрим его исходящих соседей $w$
$v \to u \to w$
Если $mark[w] == true$, то найден треугольник.

Код:
```C++
auto cmp = [&](int a, int b) {
    return g[a].size() < g[b].size() || (g[a].size() == g[b].size() && a < b);
};

sort(p.begin() + 1, p.end());
for (int v = 1; v <= n; v++)
    sort(g[v].begin(), g[v].end(), cmp);

vector<int> cnt(n + 1);
int ans = 0;

for (int i = 1; i <= n; i++) {
    int v = p[i];

    for (int w : g[v]) {
        cnt[w]++;
    }

    for (int u : g[v]) {
        if (cmp(v, u))
            break;

        for (int w : g[u]) {
            if (cmp(u, w))
                break;
            
            ans += cnt[w];
        }
    }

    for (int w : g[v])
        cnt[w]--;
}
```
